# Guia de Substituição de Imagens — Prime Web

Documento oficial do projeto. Com ele, qualquer pessoa consegue localizar,
entender e substituir qualquer imagem do site da Prime Web **sem precisar
abrir o código-fonte**.

> Todos os mockups do portfólio são capturados no próprio celular. Nenhuma
> imagem deve ser cortada — os cards já estão preparados (object-fit: contain)
> para exibir qualquer mockup vertical por inteiro.

---

## 1. Portfólio — Estudos de Caso

Seção `#trabalhos`, arquivo `portfolio.css` / `portfolio.js`. São 6 cases,
cada um com uma imagem em `portfolio-case__canvas`.

### Prime Web

- **Nome do arquivo:** `primeweb-mockup.webp`
- **Seção:** Portfólio → Case 02
- **Função:** Mockup do site institucional da própria Prime Web (case de
  reposicionamento de marca)
- **Substituir futuramente por:** Novo print de tela do site atual da Prime
  Web, capturado no celular, mantendo o mesmo conceito visual (dark premium)
- **Observações:** É o único case que documenta o próprio site da agência.
  Ao atualizar o site, este é o primeiro mockup a ser revisado.

### Studio Neiane

- **Nome do arquivo:** `studio-neiane.webp`
- **Seção:** Portfólio → Case 01
- **Função:** Mockup do site institucional do Studio Neiane Ferreira (ateliê
  de moda autoral)
- **Substituir futuramente por:** Print atualizado do site do cliente,
  mantendo a paleta clara (tecido cru / alfaiataria)
- **Observações:** Case de abertura do portfólio — deve transmitir
  minimalismo e cuidado editorial.

### Convite Digital

- **Nome do arquivo:** `convite-digital.webp`
- **Seção:** Portfólio → Case 03
- **Função:** Mockup da linha de convites digitais interativos
- **Substituir futuramente por:** Print de um convite real já entregue a um
  cliente, mostrando a tela de abertura animada ou o modal de RSVP
- **Observações:** Produto recorrente da Prime Web — vale manter sempre o
  exemplo mais recente e visualmente mais rico.

### Landing Jurídica

- **Nome do arquivo:** `landing-juridica.webp`
- **Seção:** Portfólio → Case 04
- **Função:** Mockup da landing page para escritório de advocacia
- **Substituir futuramente por:** Print atualizado da landing page do
  cliente, preservando a hierarquia tipográfica de autoridade
- **Observações:** Case usado para vender projetos de conversão (formulário
  de agendamento de consulta).

### Clínica Odontológica

- **Nome do arquivo:** `site-institucional.webp`
- **Seção:** Portfólio → Case 05
- **Função:** Mockup do site institucional da clínica odontológica
- **Substituir futuramente por:** Print atualizado do site do cliente,
  mantendo a linguagem visual mais suave (acolhimento)
- **Observações:** Nome de arquivo genérico (`site-institucional.webp`) —
  atenção redobrada para não confundir com um futuro case institucional
  diferente.

### Cardápio Digital

- **Nome do arquivo:** `cardapio-digital.webp`
- **Seção:** Portfólio → Case 06
- **Função:** Mockup do cardápio digital via QR Code
- **Substituir futuramente por:** Print do cardápio de um cliente real em
  produção, mostrando categorias navegáveis
- **Observações:** Último case da lista — fecha o portfólio com um produto
  de baixo custo de manutenção, bom gancho para o CTA final.

### Banner Marketing

- **Nome do arquivo:** `marketing.webp`
- **Seção:** Banner Premium (`#banner-promo`, "Em breve")
- **Função:** Mockup ilustrativo de painel de campanhas de marketing digital
- **Substituir futuramente por:** Arte oficial assim que o espaço de
  campanhas/lançamentos entrar no ar
- **Observações:** Único mockup em formato horizontal largo (1600×600).
  Não é um card de portfólio — é uma seção de banner reservada.

## 5. Regra geral para qualquer substituição

1. Nunca usar `object-fit: cover` em mockups de celular — todos os cards de
   portfólio já usam `object-fit: contain`, então a imagem nunca é cortada.
2. Manter o mesmo nome de arquivo ao substituir (evita precisar editar
   HTML/CSS/JS).
3. Manter o mesmo formato (`.webp`) para preservar performance de carregamento.
4. Ao trocar uma imagem, verificar visualmente se ela aparece por inteiro no
   card antes de publicar.
